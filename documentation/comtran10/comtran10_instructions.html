COMTRAN TEN -- Instruction Repertoire

COMTRAN TEN

Instruction Repertoire -- 44 Instructions -- Digiac / Fabri-Tek, 1972

Source: KDA-3032, USAF Technical Training, June 1981

How to Read This Reference

Every instruction is exactly two consecutive 8-bit bytes. The first byte is the op code. The second byte is the operand -- either an immediate value or the low byte of a memory address.

Addressing modes are indicated by suffix in the mnemonic:

,k -- immediate: operand is a constant value (0 to $FF).
,m -- memory: operand is the low byte of a memory address.
,x -- indexed: X Register is added to the memory address at execution time.

Memory map: The COMTRAN TEN has 1024 ($400) locations, addressed $000 to $3FF. These are divided into four pages of 256 bytes each:

Page 0:  $000 - $0FF
Page 1:  $100 - $1FF
Page 2:  $200 - $2FF
Page 3:  $300 - $3FF

The page number is encoded in the op code byte itself. This is the most important thing to understand when reading or writing hex by hand. The mnemonic is the same regardless of which page you address, but the hex op code changes:

LDA $0050  ->  20 50    (op $20 = LDA, page 0)
LDA $0150  ->  21 50    (op $21 = LDA, page 1)
LDA $0250  ->  22 50    (op $22 = LDA, page 2)
LDA $0350  ->  23 50    (op $23 = LDA, page 3)

The assembler computes the correct op code from the address you write. Hand-assembling means choosing it yourself. The base op code (page 0) is what the Hex Code column shows. Add 1, 2, or 3 for pages 1, 2, or 3 respectively (see note 1).

Indexed addressing sets bit 2 of the op code (adds $04 to the base). Page and index bits combine: LDA $0050,x on page 0 = op $24; on page 1 = op $25 (see note 2).

Examples show a data block at a low address, a vertical ellipsis (...), then the code block. Each instruction line shows: address, first hex byte, second hex byte, mnemonic, and a comment using <- for assignment. Where instructions share the same pattern, one example covers the group.

Load -- Load registers from memory or immediate value

LX1,k -- Load Index Immediate

Hex: $12 [3]

Load INDEX Register with a count of k.

3.Value k is limited to FF hex.
Load a register with an immediate value (pattern applies to LA1, LC1)
; Code
Addr  Hex   Hex   Mnemonic
0100  12    05    LX1,k       ; X <- $05  (load INDEX with immediate value 5)
0102  01    0A    LC1,k       ; C <- $0A  (load COUNTDOWN with 10)
0104  02    FF    LA1,k       ; A <- $FF  (load ACCUMULATOR with 255)
;
; k is always the second byte. Max value $FF. No memory access.

LC1,k -- Load Countdown Immediate

Hex: $01 [3]

Load COUNTDOWN Register with a count of k.

3.Value k is limited to FF hex.

Example pattern: see LX1 above.

LA1,k -- Load Accumulator Immediate

Hex: $02 [3, 14]

Load ACCUMULATOR with the value k.

3.Value k is limited to FF hex.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

Example pattern: see LX1 above.

LDA,m,x -- Load Accumulator

Hex: $20 [1, 2, 14]

Load ACCUMULATOR with contents of memory address m.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Load accumulator from memory -- page 0 and upper pages
; Data
Addr  Hex
0050  2F          ; value to load ($2F)
0150  7C          ; same offset, page 1

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  20    50    LDA,m       ; A <- [$0050]  op=$20 = LDA page 0
0102  21    50    LDA,m       ; A <- [$0150]  op=$21 = LDA page 1 (page in op code!)
0104  22    50    LDA,m       ; A <- [$0250]  op=$22 = LDA page 2
0106  23    50    LDA,m       ; A <- [$0350]  op=$23 = LDA page 3
;
; Same mnemonic, different hex op code. Page number is baked into
; the upper bits of the first byte. The assembler writes the right
; op code based on the address you give it. By hand, you choose it.

LCC,m,x -- Load Consecutive

Hex: $30 [1, 2]

Transfer the contents of memory address m to memory address m + 1.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Copy one memory location to the next (memory-to-memory transfer)
; Data (before)
Addr  Hex
0010  A5          ; source
0011  00          ; destination (will be overwritten)

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  30    10    LCC,m       ; [$0011] <- [$0010]  ($0011 = $A5)
;
; LCC transfers m to m+1. Accumulator is not involved.

LAN,m,x -- Load Accumulator Negative

Hex: $38 [1, 2, 14]

Load ACCUMULATOR with the two's complement of the contents of m.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Load accumulator with the two's complement (negate) of a memory value
; Data
Addr  Hex
0010  05          ; source value ($05)

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  38    10    LAN,m       ; A <- -[$0010]  (A = $FB = -5 in two's complement)
;
; Equivalent to loading then negating. Useful for setting up subtraction.

LDQ,m,x -- Load Quotient Register

Hex: $40 [1, 2]

Load QUOTIENT Register with the contents of memory address m.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Load the Quotient Register from memory
; Data
Addr  Hex
0010  00          ; high byte of dividend
0011  3C          ; low byte of dividend ($003C = 60)

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  02    3C    LA1,k       ; A <- $3C  (low byte of dividend)
0102  40    10    LDQ,m       ; Q <- [$0010]  (high byte into Q)
; AQ now holds the double-length value $003C, ready for DIV

Store -- Store register contents to memory

STA,m,x -- Store Accumulator

Hex: $48 [1, 2]

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Store a register to memory (pattern applies to STX, STQ)
; Code  (assume A = $2F, X = $0080, Q = $00)

Arithmetic -- Arithmetic operations

ADD,m,x -- Add

Hex: $60 [1, 2, 11, 12, 14]

Add the contents of memory address m to the ACCUMULATOR, leaving the result in the ACCUMULATOR. Conditionally set carry and add overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
12.ADD overflow: result exceeds largest signed 8-bit number. Numbers handled as signed two's complement.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Add memory to accumulator; subtract is the same pattern
; Data
Addr  Hex
0010  07          ; addend
0011  03          ; subtrahend

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  02    05    LA1,k       ; A <- $05
0102  60    10    ADD,m       ; A <- A + [$0010]  (A = $0C)
0104  68    11    SUB,m       ; A <- A - [$0011]  (A = $09)
; Carry set if borrow (SUB) or overflow (ADD) occurred

SUB,m,x -- Subtract

Hex: $68 [1, 2, 11, 12, 14]

Subtract the contents of memory address m from the ACCUMULATOR, leaving the result in the ACCUMULATOR. Conditionally set carry and add overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
12.ADD overflow: result exceeds largest signed 8-bit number. Numbers handled as signed two's complement.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

Example pattern: see ADD above.

MPY,m,x -- Multiply

Hex: $70 [1, 2, 14]

Multiply the contents of the ACCUMULATOR by the contents of memory address m, leaving a double length product in the AQ Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Multiply -- double-length result in AQ Register
; Data
Addr  Hex
0010  0A          ; multiplier ($0A = 10)

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  02    06    LA1,k       ; A <- $06  (multiplicand = 6)
0102  70    10    MPY,m       ; AQ <- A * [$0010]  (AQ = $003C = 60)
;
; Q holds the high byte of the 16-bit product.
; A holds the low byte.
; Result $003C: Q = $00, A = $3C

DIV,m,x -- Divide

Hex: $78 [1, 2, 13, 14]

Divide the double length number in the AQ Register by the contents of memory address m, leaving the quotient in the Q Register and the remainder in the ACCUMULATOR. Set condition code to sign of quotient. Sign of remainder equals sign of dividend. Conditionally set divide overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
13.DIVIDE overflow: quotient exceeds largest signed 8-bit number.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Divide -- quotient in Q Register, remainder in Accumulator
; Data
Addr  Hex
0010  07          ; divisor ($07 = 7)

      ...

; Code  (AQ = $003C = 60 from prior MPY, or set up manually)
Addr  Hex   Hex   Mnemonic
0100  78    10    DIV,m       ; AQ / [$0010]:  60 / 7
                              ; Q <- $08  (quotient = 8)
                              ; A <- $04  (remainder = 4)
;
; Condition code reflects sign of quotient (Q Register), not A.
; Divide overflow set if quotient exceeds signed 8-bit range.

RAO,m,x -- Replace Add One

Hex: $80 [1, 2, 4, 11, 14]

Add 1 to the contents of memory address m. If contents of m are FF, sets m to zero and sets carry bit. ACCUMULATOR contains the result.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
4.Arithmetic in RAO and RSO handles an 8-bit unsigned number.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Increment and decrement memory in place (RAO / RSO)
; Data
Addr  Hex
0010  04          ; counter

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  80    10    RAO,m       ; [$0010] <- [$0010] + 1  (= $05); A = $05
0102  88    10    RSO,m       ; [$0010] <- [$0010] - 1  (= $04); A = $04
;
; Both instructions modify memory AND load the result into A.
; RAO wraps $FF -> $00 and sets carry. RSO wraps $00 -> $FF and sets carry.

RSO,m,x -- Replace Subtract One

Hex: $88 [1, 2, 4, 11, 14]

Subtract 1 from the contents of memory address m. If contents of m are zero, sets m to FF and sets carry bit. ACCUMULATOR contains the result.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
4.Arithmetic in RAO and RSO handles an 8-bit unsigned number.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

Example pattern: see RAO above.

INX,k -- Increase Index

Hex: $03 [3]

Increase the contents of the INDEX Register by k.

3.Value k is limited to FF hex.
Increment the INDEX Register by an immediate value
; Code
Addr  Hex   Hex   Mnemonic
0100  12    00    LX1,k       ; X <- $00  (start at zero)
0102  03    01    INX,k       ; X <- X + $01  (X = $01)
0104  03    01    INX,k       ; X <- X + $01  (X = $02)
;
; INX adds k to X. To step through memory one byte at a time, use INX,1
; before or after an indexed LDA or STA.

Logical -- Logical and shift operations

SLA,k -- Shift Left Arithmetic

Hex: $0B [3]

Shift the AQ Register left k places, filling in zeros on the right.

3.Value k is limited to FF hex.
Arithmetic shifts operate on the AQ Register (double-length)
; Code  (assume AQ = $0018 = 0000 0000 0001 1000)
Addr  Hex   Hex   Mnemonic
0100  0B    02    SLA,k       ; AQ <- AQ << 2  (AQ = $0060 = 96)
                              ; shift left 2 = multiply by 4
0102  10    01    SRA,k       ; AQ <- AQ >> 1  (AQ = $0030 = 48, sign extended)
                              ; shift right 1 = divide by 2
;
; SLA/SRA work on the full 16-bit AQ pair.
; SRA propagates the sign bit (arithmetic shift).

SRA,k -- Shift Right Arithmetic

Hex: $10 [3]

Shift the AQ Register right k places, propagating sign bits on the left.

3.Value k is limited to FF hex.

Example pattern: see SLA above.

SLL,k -- Shift Left Logical

Hex: $13 [3]

Shift the ACCUMULATOR left k places, filling in zeros on the right.

3.Value k is limited to FF hex.
Logical shifts operate on the Accumulator only
; Code  (assume A = $18 = 0001 1000)
Addr  Hex   Hex   Mnemonic
0100  13    02    SLL,k       ; A <- A << 2  (A = $60 = 0110 0000)
0102  18    01    SRL,k       ; A <- A >> 1  (A = $30 = 0011 0000, zero filled)
;
; SLL/SRL work on A only, not AQ.
; SRL fills with zeros on the left (logical, not arithmetic).

SRL,k -- Shift Right Logical

Hex: $18 [3]

Shift the ACCUMULATOR right k places, filling in zeros on the left.

3.Value k is limited to FF hex.

Example pattern: see SLL above.

AND,k -- And

Hex: $19 [3]

Form the bit-by-bit logical product of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.
Bitwise AND, OR, XOR with immediate value
; Code  (assume A = $AF = 1010 1111)
Addr  Hex   Hex   Mnemonic
0100  02    AF    LA1,k       ; A <- $AF
0102  19    0F    AND,k       ; A <- A AND $0F  (A = $0F)  mask upper nibble
0104  02    AF    LA1,k       ; A <- $AF (restore)
0106  1A    50    IOR,k       ; A <- A OR $50   (A = $FF)  set bits
0108  02    AF    LA1,k       ; A <- $AF (restore)
010A  1B    FF    XOR,k       ; A <- A XOR $FF  (A = $50)  flip all bits

IOR,k -- Inclusive Or

Hex: $1A [3]

Form the bit-by-bit Inclusive OR of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.

Example pattern: see AND above.

XOR,k -- Exclusive Or

Hex: $1B [3]

Form the bit-by-bit Exclusive OR of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.

Example pattern: see AND above.

Branch -- Branch and control flow

BUN,m,x -- Branch Unconditional

Hex: $90 [1, 2]

Branch to memory address m.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Unconditional branch
; Code
Addr  Hex   Hex   Mnemonic
0100  90    10    BUN,m       ; PC <- $0010  (jump to $0010 unconditionally)
;
; BUN is the fundamental jump. BSB uses BUN internally for its return mechanism.

BST,m,x -- Branch and Stop

Hex: $98 [1, 2]

Branch to memory address m and stop.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Branch and stop -- used to pause execution at a known address
; Code
Addr  Hex   Hex   Mnemonic
0100  98    50    BST,m       ; jump to $0050 and halt
                              ; computer stops with WAIT light on
                              ; press START to continue from $0050
;
; Useful in diagnostics: stop here, inspect registers, resume.

BSB,m,x -- Branch to Subroutine

Hex: $A0 [1, 2]

Store BUN op code in location m. Store the contents of the P Register (next instruction address) in location m+1. Branch to location m+2.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Branch to subroutine -- return address stored at call site
; Code
Addr  Hex   Hex   Mnemonic
0100  A0    20    BSB,m       ; call subroutine via $0020:
                              ;   [$0020] <- $90  (BUN opcode)
                              ;   [$0021] <- $02  (return addr low: $0102)
                              ;   PC     <- $0022 (subroutine entry)
0102  ...         (execution returns here)

      ...

; Subroutine entry at $0022 (linkage occupies $0020-$0021)
0022  02    FF    LA1,k       ; subroutine body
0024  90    20    BUN,m       ; return: branch to $0020 which holds BUN $0102
;
; BSB writes its own return jump into the two bytes at the call address.
; $0020/$0021 are linkage, not code. Subroutine starts at $0022.
; Note: this mechanism means BSB is NOT re-entrant.

BPS,m,x -- Branch on Positive

Hex: $A8 [1, 2]

Branch to memory address m if the condition code is greater than zero.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Conditional branches on accumulator condition code
; Data
Addr  Hex
0010  00          ; target A value

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  20    10    LDA,m       ; A <- [$0010]
0102  A8    20    BPS,m       ; if A > 0, branch to $0020  (positive)
0104  B0    30    BZE,m       ; if A = 0, branch to $0030  (zero)

;
; Condition code set by the preceding instruction.
; Only one of the three branches will be taken.

BNC,m,x -- Branch on No Carry

Hex: $C0 [1, 2]

Branch to memory address m if the condition code is not carry.

; Code  (assume prior ADD or SUB set carry)

BXZ,m,x -- Branch on Index Zero

Hex: $C8 [1, 2]

Branch to memory address m if the INDEX Register is zero.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Count-down loop using INDEX Register and RSO
; Data
Addr  Hex
0010  05          ; loop counter (will be decremented in memory)
0020  00          ; data being processed (example)

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  12    05    LX1,k       ; X <- $05  (index for indexed addressing)
0102  20    20    LDA,m       ; A <- [$0020]  (loop body -- process data)
0104  88    10    RSO,m       ; [$0010] <- [$0010] - 1; A = decremented count
0106  B0    02    BZE,m       ; if counter = 0, branch to $0002 (done)
0108  90    02    BUN,m       ; else loop back to $0102
;

; BZE tests A (= counter) for zero. Loop runs 5 times then exits.
; INX steps the index through a data array each iteration.

SKI,k -- Skip on Interrupt

Hex: $08 [3]

Skip k instructions or 2k words if interrupt is set.

3.Value k is limited to FF hex.
Skip instructions based on hardware state (SKI / SKS / SKF)
; Code
Addr  Hex   Hex   Mnemonic
0100  08    01    SKI,k       ; if interrupt set: skip 1 instruction (-> $0104)
0102  90    10    BUN,m       ; not skipped: jump to $0010 (interrupt not set)
0104  ...         (skipped to here: interrupt was set, handle it)

0200  09    01    SKS,k       ; if sense switch ON: skip 1 instruction
0202  90    30    BUN,m       ; switch off: jump to $0030
0204  ...         (switch was on)

0300  0A    01    SKF,k       ; if flag set: skip 1 instruction
0302  90    50    BUN,m       ; flag clear: jump to $0050
0304  ...         (flag was set)
;
; k=1 skips 1 instruction (2 bytes). k=2 skips 2 instructions, etc.

SKS,k -- Skip on Sense Switch

Hex: $09 [3]

Skip k instructions or 2k words if the sense switch is on.

3.Value k is limited to FF hex.

Example pattern: see SKI above.

SKF,k -- Skip on Flag

Hex: $0A [3]

Skip k instructions or 2k words if flag is set.

3.Value k is limited to FF hex.

Example pattern: see SKI above.

Input/Output -- Input/Output and peripheral control

WDB,m,x -- Write Data Block

Hex: $D0 [1, 2, 10]

Transfer the contents of consecutive memory locations starting at address m to the selected external device. Continue until the C Register is zero. Number of words transferred is one more than the initial count in the C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
10.In HEX mode, output colons (:) are inserted automatically with line feeds and carriage returns.
Select a device then write a block of memory to it
; Data to send
Addr  Hex
0010  48          ; H
0011  45          ; E
0012  4C          ; L
0013  4C          ; L
0014  4F          ; O

      ...

; Code
Addr  Hex   Hex   Mnemonic
0100  01    04    LC1,k       ; C <- $04  (5 words: count = n - 1)
0102  11    01    OCD,k       ; select device 1 (bits 0-2 of operand byte)
0104  D0    10    WDB,m       ; write C+1 words from $0010 to device 1
;
; OCD must precede WDB/RDB to select the target device.
; C Register holds count - 1. WDB transfers count + 1 words.

MNO,m,x -- Manual Output

Hex: $D8 [1, 2, 15]

Transfer the contents of consecutive memory locations to the Input Register. Continue until the C Register is zero. Number of words transferred is one more than the initial count in the C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.

RDB,m,x -- Read Data Block

Hex: $E0 [1, 2, 7, 9]

Store data from the selected external device in consecutive memory locations starting at address m. Continue until C Register is zero. Number of words stored is one more than initial count in C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
7.RDB and RDI automatically generate and transmit XON to the teletype.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.
Read a block of data from a device into memory
; Code
Addr  Hex   Hex   Mnemonic
0100  01    09    LC1,k       ; C <- $09  (10 words)
0102  11    01    OCD,k       ; select device 1
0104  E0    20    RDB,m       ; read C+1 words from device 1 into $0020-$0029
;
; Automatically sends XON to start the tape reader (note 7).
; Stores words at $0020, $0021 ... $0029.

RDI,m,x -- Read Until Interrupt

Hex: $E8 [1, 2, 7, 8, 9]

Store data from the selected external device in consecutive memory locations starting at address m. Continue until interrupt is set.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
7.RDB and RDI automatically generate and transmit XON to the teletype.
8.Tape must end with XOFF followed by CONTROL 1.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.
Read from device until hardware interrupt is set
; Code
Addr  Hex   Hex   Mnemonic
0100  11    01    OCD,k       ; select device 1
0102  E8    20    RDI,m       ; read from device into $0020 onward until interrupt
;
; No count required -- reads until the device signals done via interrupt.
; Tape must end with XOFF then CONTROL 1 to set the interrupt (note 8).

MNI,m,x -- Manual Input

Hex: $F0 [1, 2]

Transfer the contents of the Input Register to consecutive memory locations starting at address m. Continue until the C Register is zero. Number of words stored is one more than initial count in C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Manual input -- load Input Register contents into memory
; Code
Addr  Hex   Hex   Mnemonic
0100  01    02    LC1,k       ; C <- $02  (3 words)
0102  F0    20    MNI,m       ; store Input Register -> $0020, $0021, $0022
;
; Operator loads each value via the hex keypad into the Input Register.
; MNI stores the sequence. Used for entering test data manually.

OCD,k -- Output Command

Hex: $11 [3, 6, 9]

Transmit the operand k to the external device addressed by the three low order bits in k.

3.Value k is limited to FF hex.
6.OCD instruction operand bits: bits 2,1,0 select peripheral unit; bit 3 selects HEX mode; bit 4 selects APH mode.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.
Output command -- select a peripheral device
; Code
Addr  Hex   Hex   Mnemonic
0100  11    01    OCD,k       ; select device 1 (bits 0-2 = 001)
0102  11    04    OCD,k       ; select device 4 (bits 0-2 = 100)
0104  11    08    OCD,k       ; select device with HEX mode (bit 3 = 1)
0106  11    10    OCD,k       ; select device with APH mode (bit 4 = 1)
;
; Bits 0-2 select one of 8 peripheral units.
; Bit 3 selects HEX output mode.
; Bit 4 selects APH (alphanumeric) output mode.

SST,k -- Sense Status

Hex: $00

The previously addressed device sends an 8-bit status word which is transferred to the ACCUMULATOR.

Sense status -- read device status into accumulator
; Code
Addr  Hex   Hex   Mnemonic
0100  11    01    OCD,k       ; select device 1
0102  00    00    SST,k       ; A <- 8-bit status word from device 1
;
; Operand byte is not used (note 5) but must be present (two-byte format).
; Examine A after SST to determine device ready/busy/error state.

FLC,k -- Flag Clear

Hex: $28 [5]

Clear flag to zero.

5.Operand is not used in FLAG CLEAR or FLAG SET instructions.
Set and clear the flag register
; Code
Addr  Hex   Hex   Mnemonic
0100  F8    00    FLS,k       ; flag <- 1  (set)
0102  0A    01    SKF,k       ; skip 1 if flag set (will skip)
0104  90    20    BUN,m       ; not reached
0106  28    00    FLC,k       ; flag <- 0  (clear)
;
; Operand byte not used (note 5) but required by two-byte format.
; Flag is a single programmable flip-flop, independent of condition code.

FLS,k -- Flag Set

Hex: $F8 [5]

Set flag to one.

5.Operand is not used in FLAG CLEAR or FLAG SET instructions.

Example pattern: see FLC above.

Notes

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
3.Value k is limited to FF hex.
4.Arithmetic in RAO and RSO handles an 8-bit unsigned number.
5.Operand is not used in FLAG CLEAR or FLAG SET instructions.
6.OCD instruction operand bits: bits 2,1,0 select peripheral unit; bit 3 selects HEX mode; bit 4 selects APH mode.
7.RDB and RDI automatically generate and transmit XON to the teletype.
8.Tape must end with XOFF followed by CONTROL 1.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.
10.In HEX mode, output colons (:) are inserted automatically with line feeds and carriage returns.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
12.ADD overflow: result exceeds largest signed 8-bit number. Numbers handled as signed two's complement.
13.DIVIDE overflow: quotient exceeds largest signed 8-bit number.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
15.MANUAL OUTPUT: contents of indicated memory location also displayed on Buffer Register. Computer stops with WAIT light on after each word. Press START to continue.
