COMTRAN TEN — Instruction Repertoire

COMTRAN TEN

Instruction Repertoire — 44 Instructions — Digiac / Fabri-Tek, 1972

Source: KDA-3032, USAF Technical Training, June 1981

Instruction format: two consecutive 8-bit words. First word: op code (bits 7–3) + index bit (bit 2) + page bits (bits 1–0 via note 1). Second word: operand (address low byte or immediate value k).
Addressing modes: ,m memory address   ,k immediate constant (max $FF)   ,x indexed (X Register added to address)
Memory: $000–$3FF (4 pages of 256 bytes). Page encoded in op code high bits per note 1.

Load — Load registers from memory or immediate value

LX1,k — Load Index Immediate

Hex: $12 [3]

Load INDEX Register with a count of k.

3.Value k is limited to FF hex.

LC1,k — Load Countdown Immediate

Hex: $01 [3]

Load COUNTDOWN Register with a count of k.

3.Value k is limited to FF hex.

LA1,k — Load Accumulator Immediate

Hex: $02 [3, 14]

Load ACCUMULATOR with the value k.

3.Value k is limited to FF hex.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

LDA,m,x — Load Accumulator

Hex: $20 [1, 2, 14]

Load ACCUMULATOR with contents of memory address m.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Load a value from memory
; Data
Addr  Hex   
0010  2F          ; source byte ($2F)

      ⋮

; Code
Addr  Hex   Hex   Mnemonic
0100  20    10    LDA,m       ; A <- [$0010]  (A = $2F)

Arithmetic — Arithmetic operations

ADD,m,x — Add

Hex: $60 [1, 2, 11, 12, 14]

Add the contents of memory address m to the ACCUMULATOR, leaving the result in the ACCUMULATOR. Conditionally set carry and add overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
12.ADD overflow: result exceeds largest signed 8-bit number. Numbers handled as signed two's complement.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Add memory contents to accumulator
; Data
Addr  Hex
0010  07          ; addend ($07)

      ⋮

; Code  (assume A = $05)
Addr  Hex   Hex   Mnemonic
0100  02    05    LA1,k       ; A <- $05 (immediate)
0102  60    10    ADD,m       ; A <- A + [$0010]  (A = $0C)

SUB,m,x — Subtract

Subtract the contents of memory address m from the ACCUMULATOR, leaving the result in the ACCUMULATOR. Conditionally set carry and add overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
12.ADD overflow: result exceeds largest signed 8-bit number. Numbers handled as signed two's complement.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

MPY,m,x — Multiply

Hex: $70 [1, 2, 14]

Multiply the contents of the ACCUMULATOR by the contents of memory address m, leaving a double length product in the AQ Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.
Multiply — result in AQ Register
; Data
Addr  Hex
0010  0A          ; multiplier ($0A = 10)

      ⋮

; Code  (assume A = $06)
Addr  Hex   Hex   Mnemonic
0100  02    06    LA1,k       ; A <- $06 (immediate)
0102  70    10    MPY,m       ; AQ <- A * [$0010]  (AQ = $003C = 60)
; Q Register holds high byte, A holds low byte of double-length product

DIV,m,x — Divide

Hex: $78 [1, 2, 13, 14]

Divide the double length number in the AQ Register by the contents of memory address m, leaving the quotient in the Q Register and the remainder in the ACCUMULATOR. Set condition code to sign of quotient. Sign of remainder equals sign of dividend. Conditionally set divide overflow.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
13.DIVIDE overflow: quotient exceeds largest signed 8-bit number.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

RAO,m,x — Replace Add One

Hex: $80 [1, 2, 4, 11, 14]

Add 1 to the contents of memory address m. If contents of m are FF, sets m to zero and sets carry bit. ACCUMULATOR contains the result.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
4.Arithmetic in RAO and RSO handles an 8-bit unsigned number.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

RSO,m,x — Replace Subtract One

Hex: $88 [1, 2, 4, 11, 14]

Subtract 1 from the contents of memory address m. If contents of m are zero, sets m to FF and sets carry bit. ACCUMULATOR contains the result.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
4.Arithmetic in RAO and RSO handles an 8-bit unsigned number.
11.CARRY condition code set by ADD, SUB, RAO, or RSO — indicates carry or borrow from bit 7.
14.CONDITION CODE reflects ACCUMULATOR status at end of instruction. Exceptions: (a) DIV — reflects Quotient Register; (b) MPY, SRA, SLA — reflects AQ Register.

INX,k — Increase Index

Hex: $03 [3]

Increase the contents of the INDEX Register by k.

3.Value k is limited to FF hex.

Logical — Logical and shift operations

SLA,k — Shift Left Arithmetic

Hex: $0B [3]

Shift the AQ Register left k places, filling in zeros on the right.

3.Value k is limited to FF hex.

SRA,k — Shift Right Arithmetic

Hex: $10 [3]

Shift the AQ Register right k places, propagating sign bits on the left.

3.Value k is limited to FF hex.

SLL,k — Shift Left Logical

Hex: $13 [3]

Shift the ACCUMULATOR left k places, filling in zeros on the right.

3.Value k is limited to FF hex.

SRL,k — Shift Right Logical

Hex: $18 [3]

Shift the ACCUMULATOR right k places, filling in zeros on the left.

3.Value k is limited to FF hex.

AND,k — And

Hex: $19 [3]

Form the bit-by-bit logical product of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.
Mask the accumulator (immediate)
; Code  (assume A = $AF = 1010 1111)
Addr  Hex   Hex   Mnemonic
0100  02    AF    LA1,k       ; A <- $AF (immediate)
0102  19    0F    AND,k       ; A <- A AND $0F  (A = $0F = 0000 1111)
; Masks upper nibble, preserves lower nibble

IOR,k — Inclusive Or

Hex: $1A [3]

Form the bit-by-bit Inclusive OR of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.

XOR,k — Exclusive Or

Hex: $1B [3]

Form the bit-by-bit Exclusive OR of k and the contents of the ACCUMULATOR, leaving the result in the ACCUMULATOR.

3.Value k is limited to FF hex.

BSB,m,x — Branch to Subroutine

Hex: $A0 [1, 2]

Store BUN op code in location m. Store the contents of the P Register (next instruction address) in location m+1. Branch to location m+2.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
Branch to subroutine (and return)
; Code
Addr  Hex   Hex   Mnemonic
0100  A0    20    BSB,m       ; store BUN at $0020, P+1 at $0021, branch to $0022
                              ; $0020 <- $90 (BUN opcode)
                              ; $0021 <- $02 (return address low byte: $0102)
                              ; PC -> $0022  (subroutine entry)
0102  ...         (continues here on return)

      ⋮

; Subroutine at $0020 (note: $0020/$0021 used as linkage, code starts $0022)
0022  02    FF    LA1,k       ; subroutine body -- example
0024  90    20    BUN,m       ; return: branch to [$0020] which holds BUN $0102
; The BSB mechanism stores its own return jump at the call site

Input/Output — Input/Output and peripheral control

WDB,m,x — Write Data Block

Hex: $D0 [1, 2, 10]

Transfer the contents of consecutive memory locations starting at address m to the selected external device. Continue until the C Register is zero. Number of words transferred is one more than the initial count in the C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
10.In HEX mode, output colons (:) are inserted automatically with line feeds and carriage returns.

MNO,m,x — Manual Output

Hex: $D8 [1, 2, 15]

Transfer the contents of consecutive memory locations to the Input Register. Continue until the C Register is zero. Number of words transferred is one more than the initial count in the C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
15.MANUAL OUTPUT: contents of indicated memory location also displayed on Buffer Register. Computer stops with WAIT light on after each word. Press START to continue.

RDB,m,x — Read Data Block

Hex: $E0 [1, 2, 7, 9]

Store data from the selected external device in consecutive memory locations starting at address m. Continue until C Register is zero. Number of words stored is one more than initial count in C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
7.RDB and RDI automatically generate and transmit XON to the teletype.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.

RDI,m,x — Read Until Interrupt

Hex: $E8 [1, 2, 7, 8, 9]

Store data from the selected external device in consecutive memory locations starting at address m. Continue until interrupt is set.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.
7.RDB and RDI automatically generate and transmit XON to the teletype.
8.Tape must end with XOFF followed by CONTROL 1.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.

MNI,m,x — Manual Input

Hex: $F0 [1, 2]

Transfer the contents of the Input Register to consecutive memory locations starting at address m. Continue until the C Register is zero. Number of words stored is one more than initial count in C Register.

1.To address memory locations above FF hex; 1, 2, or 3 is added to the instruction code.
2.Setting the index bit in the instruction OP CODE causes the X Register to be added to the memory address.

OCD,k — Output Command

Hex: $11 [3, 6, 9]

Transmit the operand k to the external device addressed by the three low order bits in k.

3.Value k is limited to FF hex.
6.OCD instruction operand bits: bits 2,1,0 select peripheral unit; bit 3 selects HEX mode; bit 4 selects APH mode.
9.Teletype keyboard: CONTROL H sets HEX mode; CONTROL A sets ALPHA mode.

SST,k — Sense Status

Hex: $00

The previously addressed device sends an 8-bit status word which is transferred to the ACCUMULATOR.

FLC,k — Flag Clear

Hex: $28 [5]

Clear flag to zero.

5.Operand is not used in FLAG CLEAR or FLAG SET instructions.

FLS,k — Flag Set

Hex: $F8 [5]

Set flag to one.

5.Operand is not used in FLAG CLEAR or FLAG SET instructions.
