APPENDIX
Disk Build & Distribution
Packing the demos onto a single .dsk for real hardware
DISK BUILD & DISTRIBUTION 1

Chapter 13 showed how to get one program onto a CoCo. This appendix documents the other half of the story: bundling all the demos into a single Disk BASIC image — build/demos.dsk — that you can drop onto a CoCoSDC card, serve over DriveWire, or mount in XRoar. One make target does the whole job; here is exactly what it does so you can reproduce or extend it.

What You Need — Toolshed's decb

The disk image is assembled by decb, the DECB disk utility from the Toolshed cross-development package. It creates the empty image and copies each binary in with a proper Disk BASIC directory entry.

On macOS the easiest route is Homebrew; otherwise clone and build Toolshed and put its bin/ on your PATH: 

$ brew install toolshed
# or from source: https://github.com/boisy/toolshed
#   git clone, make, then add the built bin/ to PATH
Path note: this project's decb lives at ~/bin/decb (Toolshed 2.2). If ~/bin isn't already on your PATH, run the target as PATH=~/bin:$PATH make dsk — otherwise make stops with "decb not found on PATH".

Building the Image

From the project root, one command builds the kernel, compiles every demo, and assembles the disk: 

$ make dsk

  build/demos.dsk created with 14 programs.
  Copy to SD card for FujiNet, or load in XRoar.
  In DECB:  LOADM"BOUNCE":EXEC

The dsk target depends on demos, so every .bin is rebuilt from its Forth source first (see the development workflow guide for the per-demo compile step). Then it runs, in effect: 

# 1. create a blank Disk BASIC image
$ decb dskini build/demos.dsk

# 2. copy each demo in, giving it a CoCo (8.3) name
$ decb copy src/bounce/bounce.bin build/demos.dsk,BOUNCE.BIN -2
$ decb copy src/tetris/tetris.bin build/demos.dsk,TETRIS.BIN -2
#   ... one copy line per program ...

The -2 flag selects Disk BASIC (binary machine-code) format — the same block layout LOADM expects. Each .bin already carries its kernel block, app block, and exec address, so no further processing is needed.

DISK BUILD & DISTRIBUTION 2

Verifying the Image

List the directory to confirm every program landed: 

$ decb dir build/demos.dsk

  HELLO    BIN  2  B  2
  BOUNCE   BIN  2  B  4
  CALC     BIN  2  B  4
  KALEIDSC BIN  2  B  3
  RAIN     BIN  2  B  3
  TETRIS   BIN  2  B  4
  RG-TEST  BIN  2  B  3
  TYPEWRTR BIN  2  B  3
  VDGMODES BIN  2  B  4
  CLOCK    BIN  2  B  6
  FNTIME   BIN  2  B  3
  SOUND    BIN  2  B  3
  SOUND1B  BIN  2  B  3
  SNDASYNC BIN  2  B  4

Fourteen programs, each a Disk BASIC binary (BIN, type 2). The last column is the granule count — a rough size. Names are truncated to the CoCo's 8-character limit (kaleidoscope → KALEIDSC, sound-async → SNDASYNC).

Loading on Real Hardware

The image is a standard 35-track single-sided Disk BASIC disk, so every path from Chapter 13 works — now with a whole menu of programs on one disk.

CoCoSDC / FujiNet (SD card)

  1. Copy build/demos.dsk to your SD card.
  2. Mount it as drive 0 from the SDC/FujiNet menu.
  3. At the BASIC prompt, run any program by name: 
    LOADM"TETRIS":EXEC

DriveWire 4 (serial)

  1. Point a DriveWire virtual drive at demos.dsk.
  2. Connect the USB-serial adapter to the CoCo's bit-banger port.
  3. Same load command — BASIC reads it over the wire: 
    LOADM"BOUNCE":EXEC

XRoar (emulator)

For a quick check without hardware, mount the disk and boot into Disk BASIC: 

$ xroar -machine coco2bus \
        -bas ~/.xroar/roms/bas12.rom \
        -extbas ~/.xroar/roms/extbas11.rom \
        -dos ~/.xroar/roms/disk11.rom \
        build/demos.dsk
# then, at the OK prompt:
#   LOADM"RAIN":EXEC
RTC demos: CLOCK and FNTIME read the real-time clock from a FujiNet / HDB-DOS-CC cartridge. They load and run from the disk like any other program, but only show a live time when that cartridge is present.

Single-Program Disks

The Makefile also has focused targets that build a one-program disk — handy when you want to hand someone just the clock, or the twelve tutorial examples: 

$ make tutorial-dsk   # build/tutorial.dsk — chapters 1-12
$ make clock-dsk      # build/clock.dsk
$ make fntime-dsk     # build/fntime.dsk
$ make dsks           # all of the above at once

Each follows the same dskini + copy pattern. To add a new program to demos.dsk, build its .bin and append one more decb copy line to the dsk target in the top-level Makefile.

Disk Build in a Nutshell

The Tool

Toolshed decb — DECB disk utility

~/bin/decb; put it on PATH

PATH=~/bin:$PATH make dsk

The Build

make dsk → build/demos.dsk

dskini + one copy -2 per demo

14 programs, verify with decb dir

The Load

SD card, DriveWire, or XRoar

LOADM"NAME":EXEC at the OK prompt

8-char names: KALEIDSC, SNDASYNC
