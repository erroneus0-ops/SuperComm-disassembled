Indexed Addressing — Postbyte Encoding

Every instruction using indexed addressing requires a postbyte immediately after the opcode. The postbyte encodes the pointer register, indexing type, offset size, and whether indirect addressing is used. This table is shared by all instructions that show 'indexed' mode.

Notes

  • Bits 6-5 of the postbyte select the pointer register: 00=X 01=Y 10=U 11=S
  • Bit 4 selects indirect mode (EA is address of address). Not available for auto-increment/decrement by 1.
  • For 5-bit offset (postbyte < $80): bits 4-0 are the signed offset, bits 6-5 are the register. No indirect.
  • Cycles shown are additional cycles added to the instruction's base indexed cycle count.
  • PC-relative modes use R=PC (bits 6-5 = 11 would mean S; PC-relative is a special encoding -- see below).
  • Postbyte $9F = extended indirect [addr] -- 16-bit address in the two bytes following the postbyte.

Postbyte Modes

Syntax Description Postbyte (per register) Extra bytes Extra cycles Indirect postbyte
n,R Bits 4-0 are a 5-bit signed offset (-16 to +15). Bits 6-5 select register. Bit 4 is part of the offset, NOT indirect. No additional bytes. — 0 +1 —
,R Effective address is the register value. Same as 0,R but encoded with bit 7=1. X=$84 Y=$A4 U=$C4 S=$E4 0 +0 X=$94 Y=$B4 U=$D4 S=$F4
,R+ Use register as EA, then increment register by 1. X=$80 Y=$A0 U=$C0 S=$E0 0 +2 —
,R++ Use register as EA, then increment register by 2. X=$81 Y=$A1 U=$C1 S=$E1 0 +3 X=$91 Y=$B1 U=$D1 S=$F1
,-R Decrement register by 1, then use as EA. X=$82 Y=$A2 U=$C2 S=$E2 0 +2 —
,--R Decrement register by 2, then use as EA. X=$83 Y=$A3 U=$C3 S=$E3 0 +3 X=$93 Y=$B3 U=$D3 S=$F3
n,R 8-bit signed offset (-128 to +127) in the byte following the postbyte. X=$88 Y=$A8 U=$C8 S=$E8 1 +1 X=$98 Y=$B8 U=$D8 S=$F8
n,R 16-bit signed offset in the two bytes following the postbyte (big-endian). X=$89 Y=$A9 U=$C9 S=$E9 2 +4 X=$99 Y=$B9 U=$D9 S=$F9
A,R A register (8-bit signed) added to pointer register. X=$86 Y=$A6 U=$C6 S=$E6 0 +1 X=$96 Y=$B6 U=$D6 S=$F6
B,R B register (8-bit signed) added to pointer register. X=$85 Y=$A5 U=$C5 S=$E5 0 +1 X=$95 Y=$B5 U=$D5 S=$F5
D,R D register (16-bit signed) added to pointer register. X=$8B Y=$AB U=$CB S=$EB 0 +4 X=$9B Y=$BB U=$DB S=$FB
n,PCR 8-bit signed offset from Program Counter. Used for position-independent data references. PC=$8C 1 +1 PC=$9C
n,PCR 16-bit signed offset from Program Counter. PC=$8D 2 +5 PC=$9D
[addr] 16-bit absolute address in the two bytes following the postbyte. EA is the contents of that address (indirect). Always indirect. No register involved. =$9F 2 +5 —
